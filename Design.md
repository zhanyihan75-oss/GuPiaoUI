# Design System & Style Guide
# Personal Portfolio Website

> **Reference:** https://www.victorkraemer.com (Creative Portfolio / Personal Brand)  
> **Analysis Date:** 2026-07-15  
> **Purpose:** 个人作品集网站设计系统 — 面向创意工作者 (设计师 / 开发者 / 视觉创作者)  
> **Core Vibe:** 大胆、自由、有态度、打破常规但不失专业

---

## 1. 设计概览 (Design Overview)

### 1.1 核心定位
这是一个面向 **创意工作者个人品牌** 的作品集网站。目标受众为潜在客户、招聘方、合作品牌和行业同行。

### 1.2 设计关键词
- **大胆 (Bold)** — 高饱和撞色，不畏惧视觉冲击
- **自由 (Free-spirited)** — 不规则布局，打破网格束缚
- **个性 (Personality-driven)** — 每个区块都是独立的视觉叙事
- **活力 (Energetic)** — 丰富的色彩、动效和交互层次

### 1.3 设计哲学
> *"Show, don't tell."*  
> 让作品说话，让个性溢出。不追求"完美整洁"，追求"让人记住"。  
> 这是一个有温度的个人品牌站，不是冷冰冰的企业官网。

### 1.4 与参考站的核心差异
| 维度 | 参考站 (Victor Kraemer) | 本项目 (你的作品集) |
|------|------------------------|-------------------|
| 行业 | 生活教练 / 心理疗愈 | 创意设计 / 视觉作品 |
| 色彩 | 暖金棕 + 天蓝 | 高饱和撞色 + 霓虹点缀 |
| 布局 | 对称卡片网格 | 不规则拼贴 + 错位排列 |
| 调性 | 治愈、信任 | 自信、锋芒 |
| 内容重心 | 服务介绍 + 预约转化 | 作品展示 + 个人故事 |

### 2.2 辅助色系

| 色值 | 名称 | 用途 |
|------|------|------|
| #7B2FFF | 紫罗兰 | 渐变过渡、装饰元素 |
| #00E59B | 薄荷绿 | 成功状态、次要色块 |

### 2.3 渐变策略

| 场景 | 渐变定义 | 用途 |
|------|----------|------|
| Hero 背景 | linear-gradient(135deg, #FF4D4D 0%, #FFD60A 50%, #00C2FF 100%) | 首屏视觉冲击 |
| 按钮 Hover | linear-gradient(180deg, rgba(255,255,255,0.15) 0%, transparent 100%) | 光泽覆盖层 |
| 文字渐变 | linear-gradient(90deg, #FF4D4D, #FF006E, #7B2FFF) | 标题高亮 |
| 装饰线条 | linear-gradient(90deg, #00C2FF, #7B2FFF) | 分隔线、进度条 |
| 卡片背景 | linear-gradient(180deg, #FFFFFF 0%, #F8F8FF 100%) | 微妙深度 |

### 2.4 色彩使用原则
- **每个区块可以有独立的配色** — 不必全局统一
- **但同一区块内的色彩不超过 3 种主色** — 避免混乱
- **文字始终在纯色背景上** — 不在渐变上直接放文字
- **品牌色贯穿全站** — 即使各区块配色不同，CTA 按钮保持统一的强调色

## 3. 字体系统 (Typography)

### 3.1 字体选择

| 用途 | 字体 | 字重 | 理由 |
|------|------|------|------|
| 标题 (Display) | **Clash Display** / **Space Grotesk** | 400, 500, 600, 700 | 几何感强，有态度，适合创意品牌 |
| 正文 (Body) | **Inter** / **SF Pro Display** | 300, 400, 500 | 清晰中性，不抢标题风头 |
| 代码/标签 | **JetBrains Mono** / **IBM Plex Mono** | 400 | 技术感标签、元数据 |
| 装饰文字 | **Playfair Display Italic** | 400, 500 | 引言、手写感点缀 |

> **备选方案 (全部免费):**  
> 标题: Space Grotesk (Google Fonts)  
> 正文: Inter (Google Fonts)  
> 代码: JetBrains Mono (Google Fonts)

### 3.2 字号阶梯 (Desktop)

| 层级 | 字号 | 行高 | 字重 | 字间距 | 应用场景 |
|------|------|------|------|--------|----------|
| Display XL | 72–96px | 1.0 | 700 | -2% | Hero 主标题 |
| Display L | 48–64px | 1.05 | 600 | -1% | 章节大标题 |
| H1 | 36–42px | 1.15 | 600 | 0% | 区块标题 |
| H2 | 28–32px | 1.2 | 500 | 0% | 子章节标题 |
| H3 | 20–24px | 1.3 | 500 | 0% | 卡片标题 |
| Body | 16px | 1.6 | 400 | 0% | 正文段落 |
| Body Small | 14px | 1.5 | 400 | 0% | 说明文字 |
| Caption | 12px | 1.4 | 400 | +1% | 元数据、标签 |
| Button | 14–16px | 1 | 500 | +1% | CTA 按钮 |

### 3.3 排版原则
- **标题追求冲击力** — 超大字号 + 紧凑行高 + 负字间距
- **正文追求可读性** — 适中字号 + 宽松行高
- **混合使用不同字重制造节奏** — 同一段落中穿插 bold / regular
- **全大写标题用于导航和标签** — 小字号 + 大字间距
- **文字与图像叠加时加半透明遮罩** — 确保可读性

## 5. 布局与网格 (Layout & Grid)

### 5.1 栅格系统

| 断点 | 列数 | 列宽 | Gutter | Margin |
|------|------|------|--------|--------|
| Desktop (≥1280px) | 12 | 80px | 24px | 48px |
| Laptop (1024–1279px) | 8 | 80px | 20px | 32px |
| Tablet (768–1023px) | 4 | 1fr | 16px | 24px |
| Mobile (<768px) | 4 | 1fr | 16px | 16px |

### 5.2 布局特征

#### 5.2.1 不规则拼贴 (Masonry Collage)
- 作品卡片大小不一（1×1, 2×1, 1×2, 2×2）
- 卡片之间保持固定间距（8–16px）
- 不同颜色的卡片背景形成视觉节奏
- 不按时间或分类严格排序，而是按视觉平衡排列

#### 5.2.2 错位排列 (Asymmetric Stagger)
- 左右交替的内容区块（左图右文 → 右图左文）
- 图片与文字不完全对齐，有 20–40px 的错位
- 形成 Z 型阅读路径，引导视线流动

#### 5.2.3 全屏色块 (Full-bleed Color Blocks)
- 每隔 2–3 个内容区块插入一个全屏色块
- 色块内放置一句有力的宣言或数据
- 文字居中或偏左，与色块形成强烈对比

#### 5.2.4 浮动元素 (Floating Elements)
- 装饰性图形（圆点、箭头、曲线）以绝对定位漂浮在内容之上
- 使用 z-index 层级叠加
- 在滚动时有轻微的视差效果

### 5.3 常见区块结构
`
[Hero: 超大标题 + 个人照片/插画 + 渐变背景]
[Marquee: 无限滚动的技能标签 / 合作品牌]
[Featured Work: 不规则拼贴网格]
[About Me: 左文右图 + 彩色装饰背景]
[Process: 水平时间线 / 垂直步骤卡片]
[Testimonials: 卡片轮播]
[Contact: 全屏色块 + 大号 CTA]
[Footer: 极简 + 社交链接 + 回到顶部]
`

## 6. 图像与图形风格 (Imagery & Graphics)

### 6.1 摄影风格
- **个人照片:** 自然光、生活化场景、非摆拍感
- **作品截图:** 原始比例展示，不加边框或阴影
- **背景图片:** 低饱和度或加半透明色罩，不抢前景内容

### 6.2 图形元素

| 元素 | 描述 | 使用场景 |
|------|------|----------|
| 圆点 | 实心圆，直径 8–24px，强调色 | 装饰、列表符号 |
| 箭头 | 手绘风格箭头，1–2px 描边 | 引导视线、指示方向 |
| 曲线 | 贝塞尔曲线，描边 2–3px | 分隔线、背景装饰 |
| 矩形框 | 细线框 (1px)，无填充 | 突出图片、标签背景 |
| 涂鸦标记 | 手绘下划线、圈注 | 强调重点文字 |
| 贴纸 | 不规则形状 + 阴影 | 徽章、标签、成就标记 |

### 6.3 图标风格
- **线性图标** — 1.5–2px 描边，圆角端点
- **面性图标** — 实心填充，用于导航和状态
- **混合使用** — 不同区块可以使用不同图标风格，不必全局统一
- **单色或强调色** — 图标颜色跟随所在区块的主色

## 7. 组件设计规范 (Component Guidelines)

### 7.1 按钮 (Buttons)

| 类型 | 背景 | 文字 | 圆角 | 内边距 | 特殊效果 |
|------|------|------|------|--------|----------|
| Primary | 强调色 (如 #FF4D4D) | 白色 #FFFFFF | 100px (胶囊) | 16px 40px | 悬浮时轻微放大 (scale 1.03) |
| Secondary | 透明 | 强调色 + 1px 边框 | 100px | 16px 40px | Hover 时背景变为强调色 |
| Dark | #111111 | 白色 | 100px | 16px 40px | 用于浅色背景区块 |
| Gradient | 渐变 (如 #FF4D4D → #FF006E) | 白色 | 100px | 16px 40px | 用于 Hero CTA |
| Ghost Icon | 透明 | 图标 | 50% | 12px | 社交链接、导航图标 |

**交互状态：**
- Hover: scale(1.03) + 阴影增强 + 背景色加深 5%
- Active: scale(0.98) — 按下反馈
- Disabled: opacity 0.4, cursor not-allowed
- Focus: 2px 外发光 (outline-offset: 3px)

### 7.2 作品卡片 (Project Card)

| 属性 | 值 |
|------|-----|
| 背景 | 白色 #FFFFFF 或强调色色块 |
| 圆角 | 12px |
| 内边距 | 0 (图片占满) + 下方信息区 |
| 图片比例 | 16:9 / 4:3 / 1:1 (根据作品类型变化) |
| 阴影 | 0 4px 16px rgba(0,0,0,0.08) |
| Hover 阴影 | 0 12px 32px rgba(0,0,0,0.12) |
| Hover 效果 | 图片 scale(1.05) + 轻微上浮 |

**卡片信息区：**
- 项目标题: H3 字号, 600 字重
- 分类标签: 小字号胶囊标签 (强调色背景 + 白色文字)
- 简述: Body Small, 一行截断

### 7.3 导航栏 (Navigation)

| 属性 | 值 |
|------|-----|
| 背景 | 透明 (Hero 区域) → 白色/半透明磨砂 (滚动后) |

## 8. 动效与过渡 (Motion & Transitions)

### 8.1 原则
- **有态度的动效** — 不是"平滑就好"，而是"有性格"
- **快而不躁** — 大部分动画 200–400ms
- **弹性曲线** — 使用略带弹性的缓动函数
- **滚动驱动** — 大量使用 Intersection Observer 触发

### 8.2 常用动效

| 场景 | 效果 | 时长 | 曲线 |
|------|------|------|------|
| 页面加载 | 逐元素入场 (stagger fade-in) | 600–800ms | ease-out |
| Hero 标题 | 文字逐行滑入 | 500ms | ease-out |
| 卡片 Hover | 上浮 8px + 阴影增强 + 图片放大 | 300ms | cubic-bezier(0.25, 0.46, 0.45, 0.94) |
| 按钮 Hover | scale(1.03) + 颜色加深 | 200ms | ease |
| 按钮 Active | scale(0.98) | 100ms | ease |
| 滚动触发 | 淡入 + 向上滑入 (fade-up) | 500ms | ease-out |
| Marquee | 无限水平滚动 | 持续 | linear |
| 图片加载 | 从模糊到清晰 (blur-in) | 400ms | ease |
| 数字计数 | 从 0 递增到目标值 | 800ms | ease-out |
| 导航展开 | 下拉滑入 + 菜单项 stagger | 300ms | ease-out |
| 页面切换 | 横向滑动 (slide) | 400ms | ease |

### 8.3 滚动视差 (Parallax)
- Hero 背景图以 0.5x 速度滚动
- 装饰元素以 0.3x 速度滚动
- 文字内容以 1x 正常速度滚动
- 使用 	ransform: translateY() 而非 	op 实现 GPU 加速

### 8.4 自定义光标 (可选)
- 默认光标 → 自定义圆形光标 (直径 12px)
- Hover 可点击元素时 → 光标放大 (直径 40px) + 半透明填充
- 移动时 → 光标有轻微延迟跟随 (lag)

### 8.5 避免的动效
- ❌ 超过 500ms 的简单过渡
- ❌ 没有目的的装饰性动画
- ❌ 同时触发过多元素的动画
- ❌ 移动端不使用 prefers-reduced-motion 检测

## 9. 无障碍设计 (Accessibility)

### 9.1 对比度
- 正文 ≥ 4.5:1 (WCAG AA)
- 大文本 ≥ 3:1
- **注意:** 高饱和背景上的白色文字需验证对比度

### 9.2 焦点状态
- 所有可交互元素需有可见焦点指示
- 使用 2px 强调色实线边框
- outline-offset: 2px

### 9.3 触控目标
- 最小 44×44px (WCAG 2.5.5)
- 按钮间距 ≥ 8px

### 9.4 语义化
- 正确的 heading 层级
- 图片 alt 文本
- ARIA 标签用于交互组件
- 键盘导航支持

### 9.5 运动偏好
- 尊重 prefers-reduced-motion 媒体查询
- 提供关闭动画的选项

- 渐变在深色模式下增加亮度
- 使用 CSS color-scheme: dark 原生支持

## 11. 响应式设计 (Responsive Strategy)

### 11.1 断点

| 设备 | 断点 | 主要变化 |
|------|------|----------|
| Mobile | < 768px | 单列布局，汉堡菜单，全屏色块 |
| Tablet | 768–1023px | 双列网格，缩小字号，简化装饰 |
| Laptop | 1024–1279px | 三列网格，完整导航 |
| Desktop | ≥ 1280px | 四列网格，完整动效，视差 |

### 11.2 关键响应式决策
- **作品网格:** Mobile 1 列 → Tablet 2 列 → Desktop 3–4 列
- **Hero 标题:** Mobile 48px → Tablet 64px → Desktop 80px+
- **导航:** Desktop 水平菜单 → Mobile 全屏遮罩
- **间距:** 移动端缩减 30–50%
- **图片:** 使用 srcset 提供不同尺寸

## 12. 设计检查清单 (Design QA Checklist)

### 视觉冲击力

- [ ] 色彩搭配是否有足够的视觉张力？
- [ ] 标题字号是否足够大胆？

- [ ] 精选作品 6–12 个（宁缺毋滥）

- [ ] 图片在移动端不变形、不裁切关键内容
- [ ] 卡片 Hover 有明确的交互反馈

### 一致性
- [ ] 按钮样式不超过 3 种
- [ ] 圆角半径统一（12px 为主，胶囊按钮 100px）
- [ ] 阴影层级不超过 3 级
- [ ] 字体不超过 2–3 种

### 性能
- [ ] 图片使用 WebP/AVIF 格式
- [ ] 首屏图片懒加载
- [ ] 字体使用 ont-display: swap
- [ ] 动效使用 	ransform 和 opacity（GPU 加速）

### 无障碍
- [ ] 所有颜色对比度达标
- [ ] 键盘可导航
- [ ] 图片有 alt 文本
- [ ] 尊重 reduced-motion 偏好

## 13. 设计约束与原则 (Design Constraints)

### 13.1 代码简洁性优先

| 原则 | 具体要求 | 反例 |
|------|----------|------|
| 低复杂度 | 组件逻辑不超过 100 行，函数不超过 20 行 | 一个组件里写 500 行逻辑 |
| 内容可替换 | 图片、文字、视频通过配置文件或 props 注入，不硬编码 | 直接在 JSX 里写 <img src="C:\..." /> |
| 零运行时 JS（优先） | 能用 CSS 实现的动画/布局，不用 JavaScript | 用 JS 实现一个 hover 效果 |
| 无过度抽象 | 不造不必要的 hook、不建不必要的层级 | 为一个按钮封装 3 层高阶组件 |

### 13.2 复用性标准

- **每个组件必须至少有 2 个使用场景** 才会被抽取为独立组件
- **共用样式通过 CSS Variables 管理**，不在每个组件里重复写相同的值
- **图片和媒体资源走统一配置**（content/config.json 或类似文件）
- **不为了"未来可能用到"而提前抽象**（YAGNI 原则）

### 13.3 风格一致性强制规则

- **任何实现细节不得违背本节定义的设计规则**
- 当布局、字体、颜色等实现选择与设计系统冲突时，**以设计系统为准**
- 不允许在代码中出现未在 Design.md 中定义的色值、字号、间距
- 所有新组件必须先对照本设计系统检查，再通过后再合入

### 13.4 技术选型约束

| 类别 | 推荐 | 禁止/避免 |
|------|------|----------|
| 动画 | CSS transitions / keyframes, Framer Motion (仅复杂场景) | GSAP (过重), 自定义 JS 动画库 |
| 布局 | Flexbox, CSS Grid | Float, table 布局 |
| 图标 | SVG inline 或 Lucide/Phosphor (轻量) | FontAwesome (体积大) |
| 图片 | WebP + placeholder, 
ext/image 或 Astro Image | Base64 内嵌大图 |
| 字体 | Google Fonts (subset 加载) | 自托管完整字体文件 |
| 状态管理 | 组件局部 state, Context (仅全局主题) | Redux, Zustand (过度) |

---## 13. 设计决策记录 (Decision Log)

### Q1: 为什么选择高饱和撞色而非低饱和莫兰迪色？
**A:** 个人作品集的核心目标是"让人记住你"。低饱和配色虽然安全，但容易淹没在众多设计师网站中。高饱和撞色传递自信、创造力和不妥协的态度——这正是创意工作者需要向雇主和客户传达的品牌信号。

### Q2: 为什么布局允许不规则拼贴而非严格网格？
**A:** 严格的网格虽然整洁，但会削弱个人品牌的"人味"。不规则拼贴模拟了创意工作者的思维跳跃感——既有结构又有自由。关键是保持 8px 的基础间距系统作为隐性秩序，让"乱"中有"序"。

### Q3: 为什么按钮选择胶囊形状 (100px 圆角)？
**A:** 胶囊按钮是当代创意网站的标志性元素之一。它传递友好、现代、不拘谨的感觉。相比直角或 4px 小圆角的严肃感，100px 的全胶囊圆角在视觉上更有亲和力，同时保持了足够的存在感。

### Q4: 为什么标题使用 Space Grotesk 而非更常见的 Inter/Poppins？
**A:** Inter 和 Poppins 虽然好用，但已经过于普遍。Space Grotesk 保留了几何无衬线体的清晰可读性，同时在某些字母造型上有独特的复古未来感（如 'a' 和 'R' 的造型），这正好契合创意工作者的个性表达需求。

### 13.5 实现优先级（从高到低）

1. **正确性** — 视觉效果与设计稿一致
2. **可维护性** — 内容可替换，结构清晰
3. **性能** — 加载快，运行时流畅
4. **代码优雅** — 在不牺牲前三者的前提下追求简洁

> **核心信条：** 一个 50 行的清晰组件 > 一个 200 行的"完美"组件。  
> 作品集网站的生命周期是 1–3 年，期间你会反复修改内容。**简单就是最大的高级。**

### B. 推荐图片优化
- 格式: WebP (首选) / AVIF / JPEG (fallback)
- 尺寸: 按断点提供 3 档 (480px, 768px, 1280px)
- 懒加载: 非首屏图片使用 loading="lazy"
- 占位: 使用 placeholder="blur" 或 CSS 骨架屏

### C. 推荐 CSS 框架
- **Tailwind CSS** — 高度自定义，适合创意布局
- **UnoCSS** — 更快的原子化 CSS 引擎
- **原生 CSS + CSS Variables** — 最轻量的选择

### D. 推荐技术栈 (根据你的技术水平)
- **静态站点:** Astro + Tailwind (推荐，零 JS 负担)
- **React 生态:** Next.js + Tailwind + Framer Motion
- **极简方案:** Hugo / Jekyll + 原生 CSS
- **可视化搭建:** Framer / Webflow (适合不想写代码的阶段)
